## 2. 사용자 의도 (확정본)

### 영상 처리 정책
1. 얼굴 전용 영상 → 스킵 (자막은 유지)
2. PPT 1번이라도 나오면 → 포함
3. PiP (작은 슬라이드 + 강사 얼굴) → **슬라이드로 간주, 추출 대상**
4. 같은 슬라이드 + 필기 진행 → **필기 없는 선명한 첫 화면 1장만** (사용자 명시)
5. 결과 PPTX → `{video}.lecture.pptx` 영상 옆에 저장

### "왜 첫 화면만?"
강사 필기는 학습 보조일 뿐 본 PPT 원본은 동일. **PPT 원본 그대로 재생산**이 목표 → 가장 깨끗한 시점 1장.

## 4. 기능 요구사항

### F1. 얼굴/슬라이드 자동 분류 (전부 로컬, 무료)

**분류 카테고리**: `slide` / `face` / `mixed` (PiP)

**알고리즘 (모두 로컬, API 호출 0):**
1. **얼굴 검출**: OpenCV haarcascade (또는 macOS Vision API의 얼굴 검출)
2. **텍스트 영역 비율**: OpenCV Canny edge + contour 분석 — PPT는 텍스트 박스 많음
3. **색 균일도**: HSV histogram — PPT 배경은 단색·그라데이션, 얼굴은 피부톤 dominant
4. **규칙 결합**:
   - 얼굴 면적 > 30% & 텍스트 영역 < 10% → `face`
   - 얼굴 면적 < 15% & 텍스트 영역 > 30% → `slide`
   - 둘 다 있음 → `mixed` (PiP — 슬라이드로 처리)
5. **PiP crop**: mixed로 분류된 프레임은 강사 얼굴 영역 자동 mask 후 슬라이드 영역만 남김 (사용자 정책)

**영상 전체 판정**: slide + mixed 프레임 0개 → 얼굴 전용 영상으로 스킵

### F2. 중복 슬라이드 제거 (필기 없는 첫 화면 1장 정책)

**캡쳐 → 클러스터링 → "첫 깨끗한 화면" 선정**

1. **ffmpeg scene detection** (Rev.3 재사용): `select=gt(scene,0.25)`
2. **pHash 클러스터링**: 캡쳐된 PNG들을 시각 유사도로 묶음 (유사도 ≥ 0.85)
3. **클러스터 안에서 "필기 없는 첫 화면" 선정**:
