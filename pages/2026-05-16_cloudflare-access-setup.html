Cloudflare Access 셋업 가이드

Private 리포 자료를 본인 인증 후 외부 URL로 어디서나 열기.
이메일 OTP 1회 인증 → 7일간 자동 접근.

⏱ 예상 소요: 25~35분 (Cloudflare 가입 + 셋업)

0이 가이드가 만들어낼 최종 상태

💡 원리 Cloudflare Pages가 Public 리포처럼 정적 파일을 배포하되, Cloudflare Access(Zero Trust)가 그 URL 앞에 인증 게이트를 세웁니다. 사용자가 본인 이메일로 OTP 인증해야만 사이트가 보입니다.

1Cloudflare 계정 준비 (5분)

STEP 1.1
브라우저에서 https://dash.cloudflare.com/sign-up 접속
→ 이메일 celab@snapscale.io + 비밀번호 입력 → "Sign Up"
STEP 1.2
이메일 본인 인증 — Cloudflare가 보낸 메일에서 "Verify Email" 클릭
✓ 이미 Cloudflare 계정이 있다면 이 단계를 건너뛰고 1.3으로 가세요.
STEP 1.3
Cloudflare 대시보드 로그인 → 좌측 메뉴 확인 → "Workers & Pages"와 "Zero Trust" 두 항목이 보이는지 확인. (안 보이면 페이지 새로고침)

2Cloudflare Pages 프로젝트 생성 (10분)

STEP 2.1
좌측 "Workers & Pages" 클릭 → 상단 "Create" → "Pages" 탭 → "Connect to Git"
STEP 2.2
"GitHub" 선택 → GitHub 인증 팝업이 뜨면 celab-kor 계정으로 로그인 → "Authorize Cloudflare" 클릭
STEP 2.3
"Select a repository" 화면 → "celab-wiki" 선택 → "Begin setup"
(celab-knowledge-base도 같이 하고 싶다면 2단계 반복. 한 번 권한 부여하면 양쪽 모두 가능)
STEP 2.4 — 빌드 설정
다음과 같이 입력:
Project namecelab-wiki
Production branchmain
Framework presetNone
Build command(비워둠)
Build output directory/
→ 하단 "Save and Deploy" 클릭
STEP 2.5 — 배포 완료 확인
약 1~2분 후 배포 완료. "Visit site" 버튼이 나타남.
자동 생성된 URL은 https://celab-wiki.pages.dev 형식 (또는 celab-wiki-xyz.pages.dev).
→ 이 URL을 메모해두세요. Step 4에서 사용합니다.
⚠️ 지금 단계에서는 누구나 접근 가능 Pages 배포만으로는 인증이 없어 누구나 URL로 접근 가능합니다. Step 3 Cloudflare Access 셋업 전까지 URL을 누구에게도 공유하지 마세요.

3Cloudflare Access (Zero Trust) 셋업 (10~15분)

STEP 3.1 — Zero Trust 활성화
좌측 메뉴 "Zero Trust" 클릭 → 첫 진입 시 "팀 이름(Team name)" 입력하라고 함:
예시: celab 또는 snapscale입력 → 무료 플랜 (Free) 선택 → 결제 정보 없이 진행 가능 → "Next"
📌 팀 이름의 의미 팀 이름은 {팀이름}.cloudflareaccess.com 같은 인증 도메인이 됩니다. 이메일 OTP 인증 페이지의 URL 일부예요. 한 번 정하면 변경 어려우니 짧고 기억하기 쉽게.
STEP 3.2 — Access Application 생성
Zero Trust 대시보드 → 좌측 "Access" → "Applications" → 상단 "Add an application" → 카드 중 "Self-hosted" 선택
STEP 3.3 — Application 설정
다음과 같이 입력:
Application nameCELab Wiki
Session Duration24 hours (또는 원하는 만큼; 7 days 권장)
Application domaincelab-wiki.pages.dev (Step 2.5에서 메모한 URL)
→ 하단 "Next"
STEP 3.4 — 인증 정책 (Policy)
"Configure rules" 화면:
Policy nameAllow celab@snapscale.io
ActionAllow
Session durationSame as application

"Configure rules" 섹션에서:

  • Selector: Emails
  • Value: celab@snapscale.io 입력 후 Enter
→ 우측 하단 "Next"
STEP 3.5 — Identity Provider (인증 방식)
기본으로 "One-time PIN"이 체크되어 있어야 함. (이메일로 6자리 OTP 받는 방식)
만약 안 되어 있으면 체크.
→ 하단 "Next" → 다음 페이지(고급 설정) 그대로 → "Add application"
✅ 여기까지 완료하면 이제 celab-wiki.pages.dev에 접속 시 Cloudflare Access 인증 페이지가 먼저 뜨고, 이메일 입력 → OTP 발송 → 코드 입력 후에만 사이트가 보입니다.

4인증 흐름 테스트 (5분)

STEP 4.1
새로운 시크릿/프라이빗 창을 열고 (Chrome: ⌘+Shift+N / Safari: ⌘+Shift+N) → https://celab-wiki.pages.dev 접속
STEP 4.2
Cloudflare Access 로고와 함께 이메일 입력 화면이 떠야 정상.
celab@snapscale.io 입력 → "Send me a code" 클릭
STEP 4.3
이메일(celab@snapscale.io)을 확인 → Cloudflare가 보낸 6자리 코드 복사 → 페이지로 돌아가 입력 → "Continue"
STEP 4.4
CELab Wiki 갤러리가 떠야 정상. ✅ 셋업 완료.
이제 24시간(또는 설정한 기간) 동안 같은 브라우저에서 자동 인증됩니다.
❌ 실패 시 체크리스트
  • Policy의 이메일 주소가 celab@snapscale.io와 완전히 일치하는지 (오타 검사)
  • Application domain이 Pages URL과 일치하는지
  • One-time PIN이 활성화되어 있는지
  • Pages 빌드가 성공했는지 (Pages 대시보드의 Latest deployment 확인)

5(선택) 같은 방식으로 celab-knowledge-base도 보호

기존 지식창고도 같은 방식으로 처리하려면 Step 2 + Step 3을 반복하세요:

  1. Step 2 — Cloudflare Pages에 celab-knowledge-base 리포 추가 배포
  2. Step 3 — Access Application "CELab Knowledge Base" 생성, 도메인 celab-knowledge-base.pages.dev로
💡 한 번 Policy 만들었으면 재사용 가능 Step 3.4에서 만든 "Allow celab@snapscale.io" Policy는 새 Application 만들 때 그대로 재사용할 수 있어 두 번째부터는 빠릅니다.

6셋업 완료 후 URL을 저에게 알려주세요 — 자동 처리

Step 2.5에서 생성된 URL 1~2개를 알려주시면 다음을 자동으로 진행합니다:

  • 슬랙 발송 헬퍼(scripts/notify_slack.py)의 KB_BASE_URL을 새 URL로 변경
  • 지식창고열기.app의 AppleScript URL 업데이트
  • Raycast Quicklink "지식창고" URL 업데이트 (수동 1클릭 필요)
  • 발행대기처리.command가 슬랙에 보내는 링크도 자동으로 새 URL 사용
  • 매뉴얼 HTML(2026-05-15_user-manual.html) 안의 갤러리 링크 업데이트
📩 셋업 끝나면 이렇게 알려주세요 
"celab-wiki URL = https://celab-wiki.pages.dev
celab-knowledge-base URL = https://celab-knowledge-base.pages.dev"
이 두 줄만 주시면 나머지 모두 자동 처리합니다.
