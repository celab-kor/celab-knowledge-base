SESSION 01

EPC 산업 구조와 KPI 이해

화공플랜트 EPC(Engineering, Procurement, Construction) 사업의 기본 구조와 사업자가 최우선으로 추구하는 KPI(핵심 성과지표)를 설명한 세션.

  • EPC사는 발주처로부터 제한된 예산과 기간을 부여받아 경쟁 입찰(Competitive Bidding)로 수주 → 구조적으로 시간·비용 항상 부족
  • EPC의 제1 KPI = 내부 맨아워(Man-Hour) 절감 — 설계부터 시공까지 전 단계에 걸쳐 적용
  • 발주처 → EPC → 벤더(장비/배관 제조) → 시공업체 순의 수직적 외주 체계
  • SnapScale이 공략하는 핵심 페인 포인트: 설계 단계에서의 맨아워 낭비 (전체 프로젝트 비용의 5~8%)
SESSION 02

EPC 수주 프로세스 상세 해부

발주처의 ITB 발행부터 최종 계약 체결까지, 실무 수주 단계의 흐름과 각 단계별 리스크를 Q&A 형식으로 심층 논의한 세션.

  • ITB(Invitation to Bidding) 수신 → 1주일 내 프로포절(견적) 제출 → 우선협상대상자 선정 → 최종 계약
  • 초기 견적 기준: 라이선서가 제공하는 FEED(기본 설계)의 30~50% 마진 내 금액에서 각 EPC사가 최대한 타이트하게 경쟁
  • 영업이익률 3~5% 구조에서 한 건의 설계 오류가 전체 마진을 날릴 수 있는 구조
  • 우선협상대상자 이후에도 계약 취소 사례 다수 — 요구사항 미반영, 금액 협상 결렬 등
  • 리스크 관리: PME 각 부서별 안전마진 → 임원 안전마진 → 컨틴전시(Contingency) 이중·삼중 구조
SESSION 03

설계 딜리버러블(성과품) 체계와 화공 장비 구성

EPC 설계 업무의 핵심 산출물인 딜리버러블들의 종류와 역할, 그리고 화공플랜트를 구성하는 주요 공정 장비들을 체계적으로 설명한 세션.

  • BED(기본 설계 근거) → BFD(블록 플로우 다이어그램) → PFD(공정 흐름도) + HMB(열/물질 수지) → P&ID(배관계장도) → 아이소메트릭 → 3D 모델링 순서로 설계 전개
  • P&ID가 모든 부서의 설계 기준점 — 공정·배관·장비·계장·전기 전 팀이 P&ID를 중심으로 작업
  • 리스트(Line List, Equipment List, Instrument List) 작성이 가장 반복적 수작업 — P&ID 100장당 배관 1,000개 이상 수동 작성
  • 주요 장비: 펌프(압력 유지), 밸브(흐름 차단), 열교환기(온도 조절), 리액터(화학반응), 베셀/타워(유체 처리), 컴프레서(가스 압축)
  • HAZOP 리포트: 폭발·화재 시나리오별 산업 표준 기반 위험성 평가 문서
  • 심볼 레전드: 프로젝트마다 커스텀 팔레트 사용 — AI 인식의 핵심 난관
SESSION 04

SnapScale 핵심 기술 아키텍처와 제품 철학

SnapScale이 개발 중인 AI 자동화 툴의 기술 스택, 동작 원리, 그리고 엔지니어 맨아워 절감을 핵심 KPI로 설정한 제품 철학을 공개한 세션.

  • DexPI 온톨로지 개량 버전: 고정된 어트리뷰트를 동적으로 변환, 프로젝트별 커스터마이징 대응
  • 벡터 기반 P&ID 인식 → 최종 픽셀/딥러닝 비전 모델 보조 방식으로 발전 예정
  • Google ADK 활용: LLM에 함수 툴셋 제공 → 결정론적 동작 + 동작 추적 가능성 확보
  • Self-Feedback Loop + Human-in-the-Loop: AI 자동화 정확도의 하방 보장 + 엔지니어 최소 개입으로 정확도 업사이드 확보
  • 최우선 KPI = 맨아워 절감 (정확도는 세컨더리 지표) — "100시간을 50시간으로"가 핵심
  • 설계 비용에서 라이선스 피로 4%까지 수익화 가능성 제시
  • 고객 이해(도메인 지식)의 지속적 업데이트를 팀 핵심 역량으로 설정

📊 프로포절 산출 구조 — 마진의 마진 안전마진 체계 ▼
각 부서(설계팀, 조달팀, 시공팀) → 부사장 레벨 안전마진 추가 → 임원 레벨 컨틴전시 추가 → 최종 제출 금액. 이 중층 구조로 리스크를 흡수하나, 경쟁 입찰 특성상 너무 높은 금액은 수주 실패로 연결. 결국 엔지니어링 능력(설계 불확실성 조기 파악)이 마진의 핵심 결정 요인.
📁 딜리버러블(성과품) 검수 사이클 ▼
설계 전 기간에 걸쳐 약 3~4번의 대형 검수 사이클 진행. 각 사이클에서 발주처·라이선서·EPC 전 부서가 참여해 코멘트 발행 → 각 팀 반영 → 다음 사이클 검수. 이 과정에서 부서 간 핑퐁 수정이 반복되며 맨아워가 누적 소모.
⚠️ 펀치리스트(Punch List) — 이익을 갉아먹는 숨은 위험 ▼
현장 시공 중 발견되는 개별 오류 아이템. 설계·조달·시공·시운전·O&M 전 단계가 연결되어 있어 하나의 펀치리스트가 연쇄적 영향을 미침. 영업이익 3~5%짜리 프로젝트에서 펀치리스트 누적이 전체 마진 소멸의 직접 원인이 됨.
📄 핵심 성과품(Deliverables) 체계

설계 단계에서 생성되는 주요 문서 및 도면의 역할 구조

📘

BED (Basic Engineering Design Basis)

전체 공장의 건설 전제조건 총망라. 건설 지역·기후, 원자재 사양, 목표 생산량, 라이선서·소프트웨어 스펙까지 초기 조건 전체를 정의.

설계 시작점 기준 문서
🔷

BFD → PFD + HMB

블록 수준에서 시작해 실제 공정 유닛 조합으로 상세화. HMB(열·물질 수지)가 PFD와 쌍을 이루어 각 스트림의 조성·온도·압력을 수치로 고정.

물질/에너지 밸런스의 핵심
⚡

P&ID (Piping & Instrumentation Diagram)

모든 설계의 중심 문서. 배관·계장·밸브·장비의 상세 연결을 표시. 전 부서(공정·배관·장비·계장·전기)가 이 도면을 기준으로 설계 진행.

설계 업무의 표준 기준점
📊

각종 리스트 (Line/Equipment/Instrument)

P&ID에서 추출하는 배관·장비·계장 목록 엑셀. 현재까지도 수작업으로 작성. P&ID 100장 기준 배관 1,000개+ 수동 입력 → 휴먼에러의 온상.

SnapScale 1차 자동화 타깃
📐

아이소메트릭 + 하이드롤릭 시트

2.5D 배관 도면으로 꺾임 횟수·길이를 산출해 압력 강하를 계산. 엑셀 매크로 기반 계산이나 값 입력 오류·매크로 오류 위험 상존.

압력 계산 핵심 도구
🛡️

HAZOP 리포트 & 데이터 시트

폭발·화재 위험 시나리오를 산업 표준(한국/미국)에 따라 분석. 데이터 시트는 개별 장비의 두께·반지름·최대 온도압력 등 상세 스펙 문서.

안전 검증의 핵심 근거
💡 주요 개념 인사이트

세미나에서 도출된 핵심 비즈니스·기술 인사이트

⏱️

설계 비용의 절반이 낭비

전체 프로젝트 비용 중 설계가 차지하는 비중은 5~8%. 이 중 50% 이상이 수작업 반복·휴먼에러·핑퐁 수정으로 낭비. SnapScale의 직접 공략 타깃.

설계 비용 절감 잠재 = 2~4% of 전체 프로젝트
🎯

KPI는 맨아워, 정확도는 세컨더리

100시간 작업을 200시간으로 만드는 AI는 실패. 50시간으로 줄이는 게 목표. 정확도는 Human-in-the-Loop으로 지속 보강하는 구조로 접근.

Primary KPI = Man-Hour 절감
🔗

온톨로지가 변경 전파의 핵심

도면 100장의 정보를 단일 지식 체계로 묶으면, 하나 수정 시 연결된 모든 도면에 변경이 전파 가능. Git의 의존성 관리와 유사한 개념.

동시성·의존성 문제 해결
🛡️

보험 성격의 제품 가치

설계 오류 하나가 전체 마진을 날리는 구조에서, SnapScale은 단순 비용 절감을 넘어 컨틴전시 항목 자체를 줄여주는 보험 역할 수행 가능.

리스크 헤징 + 비용 절감 이중 가치
🧠

도메인 지식 격차 = 최대 리스크

20년 경험의 엔지니어가 보유한 암묵지를 AI 팀이 몇 개월 공부로 따라잡는 건 불가능. 고객(엔지니어)과의 지속적 협력이 제품의 실질적 경쟁력.

고객 이해 지속 업데이트 필수
📊

결정론적 AI 동작의 설계

💼 투자자 시각

시장 규모·수익 구조 분석

  • 글로벌 EPC 시장은 수조 원 단위 단건 프로젝트 — 설계 비용 절감의 절대값이 매우 큼
  • 설계 비용(5~8%) × 낭비율(50%) = 프로젝트당 2.5~4% 절감 잠재로 ROI 명확
  • 라이선스 피 4% 수익화 모델 제시 — SaaS 구독보다 프로젝트 단위 계약이 현실적
  • 현재 실증 사례 부재가 최대 리스크 — 파일럿 계약의 성공 여부가 밸류에이션 결정
  • 개런티 모델 불가 — 인과 관계 입증의 어려움, 수익화 구조 고도화 필요
  • 영업이익률 3~5% 산업에서 AI 도입 효과의 정량화 방법론 개발 필수
🚀 VC 시각

성장성·경쟁 해자 분석

  • AspenTech 등 기존 소프트웨어는 툴 제공만 — 컨설팅·개런티 영역 공백이 SnapScale 기회
  • 도메인 지식 + AI 기술의 결합이 진입장벽 — 복제하기 어려운 지식 자산
  • 온톨로지 기반 지식 체계가 누적될수록 네트워크 효과 발생 — 데이터 해자
  • 설계 → 조달 → 시공 순차적 확장 가능한 플랫폼 구조 잠재
  • 실증 사례 확보 후 글로벌 EPC 메이저(현대건설, 삼성엔지니어링 등) 공략 시나리오 유효
  • Human-in-the-Loop 구조가 오히려 신뢰성 있는 엔터프라이즈 도입 명분 제공
🛠️ 창업가 시각

실행 과제·제품 전략

  • 1차 타깃: 설계 리스트 자동화(Line List, Equipment List) — 즉시 현금화 가능한 페인 포인트
  • 파일럿 계약사와 실증 → 정량적 맨아워 절감 데이터 확보가 다음 영업의 핵심 자산
  • AI 작업 과정의 시각화(사고 과정 표시, 변경 하이라이팅)가 엔드유저 신뢰 확보의 핵심
  • 성능·동시성 이슈 사전 설계 필수 — 실제 계약 후 O&M 단계에서 다수 이슈 발생 예상
  • 고객 엔지니어를 파트너로 만드는 태도 — "모르면 물어보는" 스탠스가 도메인 진입 전략
  • DexPI 온톨로지 개량판 + Google ADK + 벡터 P&ID 인식 기술 스택의 조기 안정화 필요

컨틴전시 Contingency 예비 비용. 설계 오류·리스크 흡수용으로 각 단계에서 마진 추가
HAZOP Hazard and Operability Study 폭발·화재 등 위험 시나리오 분석 리포트. 산업 표준 기반

⚙️ SnapScale 핵심 기술 스택

화공플랜트 설계 자동화를 위한 기술 아키텍처

🧬

DexPI 온톨로지 (개량판)

프로젝트별 동적 어트리뷰트 지원. 고정된 표준 구조의 한계를 극복한 커스텀 지식 체계

👁️

벡터 기반 P&ID 인식

PDF/CAD의 직선·원·곡선 조합을 분석해 심볼 자동 인식. 최종 딥러닝 비전 모델로 고도화 예정

🤖

Google ADK (에이전트)

함수 툴셋 제공 → LLM의 결정론적 동작 구현. 시퀀스·루프·조건식 기반 에이전트 설계 가능

🔄

Self-Feedback Loop

인식 오류 자동 감지 → 재인식 반복. 정확도 하방을 고정하고 예외 케이스 자율 처리

👤

Human-in-the-Loop UX

AI 수정 사항 하이라이팅 + 수락/거절 버튼. 엔지니어 인지 부하 최소화로 개입 효율 극대화

🔍

SHACL + SPARQL

온톨로지 인스턴스 검증·쿼리. 자연어 조건을 그래프 수준 검증 규칙으로 자동 변환

🖥️ 실제 데모 구조 (투자자 시연 버전)

로그인 → 공유 폴더 연동 → P&ID 도면 업로드 → 심볼 레전드 팔레트 매핑 → 온톨로지 자동 구조화 → Line/Equipment/Instrument List 자동 생성(CSV) → 미인식 심볼 그레이 박스 하이라이팅 → 아이솔레이션 필로소피 대조 검증 → 오류 보라색 표시. VSCode 유사 3분할 UI(파일트리 | 도면 뷰어 | AI 채팅).
