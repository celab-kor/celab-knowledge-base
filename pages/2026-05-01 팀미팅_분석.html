💡 핵심 요약

📋 안건 내용 상세

  • 계약금 1억 9천만원 명시, 한 달 이내 입금 예정

  • 다음 인터뷰 대상: 소방, HVAC, 구매 부서 (다음 주 예정)
🔍 견적 자동화 — 설계 전주기 자동화의 '초성' 역할

이번 인터뷰를 통해 발견된 핵심 인사이트:

  • 견적팀이 전체 스케줄 수립 → 각 딜리버러블 물량을 각 부서에 배분
  • 각 부서(공정팀, 배관팀 등)가 P&ID 장수·라인 수·계장 수 등을 산출
  • 산출 방식: 레퍼런스 기반 추정 또는 직접 수량 산정
  • 결론: 견적 업무에서도 부분적 도면 인식(라인 수, 장비 수, 계장 수 카운팅)이 필수 → 도면 해석 기능이 자연스럽게 포함됨
  • 이로써 견적 자동화 ⊃ 설계 자동화의 서브셋이라는 구조 확인
  • 필즈 대표 요청 사항이자, 연구소장(이종혁 원장)의 '기본 설계 전중기 자동화' 추진과 맞닿아 있음
💬 이종혁 원장 코멘트 — 견적 자동화 평가

견적은 정확도보다 스피드가 핵심. AI가 사람보다 압도적으로 빠르므로 충분한 의미가 있음. 논리적 견적 과정을 설명할 수 있으면 충분. 상세 설계 단계의 고정밀 요구와는 다른 기준 적용 가능.

  • 1년 프로젝트 = 제품 검증 기회: 대기업 직행 대신 필즈 규모에서 충분히 검증 후 대형 계약 레퍼런스화 가능

🔬안건 6 — 엔지니어링 기술 질의응답 (이종혁 원장)
⚙️ 하이드롤릭(Hydraulic) 계산 Q&A — 다솜
  • 구간 분할 기준: 고도(Elevation) 기준 가장 높은 지점에서 구간 분할 → 확인 완료
  • 밸브·FE·FV 처리: Control Valve, Flowmeter는 등가길이(Equivalent Length) 산정 불가 → 별도 ΔP를 다음 구간 Operation Pressure에서 직접 차감하는 방식으로 처리
  • 수치 불일치(휴먼 에러): 고도 차이(1780mm)와 파이프 치수 합산(1920mm) 약 140mm 불일치 → 14~15cm 수준으로 무시 가능한 휴먼 에러로 확인
  • T-피팅 BOM 구간 배분: 구간 경계에 걸친 T, 엘보우는 아무 구간에 포함해도 무방. ISO 도면에 표현된 구간에 넣는 것이 가장 명확
  • 점선 피팅: 점선으로 표현된 피팅 = 해당 페이지에서는 미고려. 다른 페이지에서 반영됨
  • 벤더 프린트 ΔP: 벤더 데이터 시트에서 ΔP 확인 후 해당 구간 시작 압력에서 차감. 양이 많아 전부 처리 불필요 (메모 수준)
🔧 PSV(압력안전밸브) API 문서 해석 — 이종혁 원장

API 문서 조동사 의미 (강도 순):

  • SHALL: 지구가 망해도 반드시 수행. 협의 불가. 가장 강한 의무
  • MUST: 반드시 수행. SHALL과 유사하나 약간 하위
  • SHOULD: 해야 함. 원칙적 의무
  • MIGHT/MAY: 협의 가능. 하는 것이 좋으나 상황에 따라 미수행 가능

💡 SHALL이 가장 강한 의무 — 예상과 반대!

📐 전기·계장 부서 인터뷰 준비 — 필즈 다음 주 인터뷰

이미 파악된 용어:

  • Electrical Load List, Single Line Diagram (전기팀)
  • P&ID, Instrument List, Valve List (계장팀)

다음 주 인터뷰 대상 부서:

  • 소방 (Fire Protection): 스프링클러, 소화설비 물량 산출
  • HVAC: Heat, Ventilation, Air Conditioning — 실내 온·습도 제어 설비
  • 공사 (Construction): 시공 최근접 영역 (Tong-up 건축 인터뷰에서 확인)
  • 구매 (Procurement): MR(Material Requisition), LD(Liquidated Damages) 등 계약 조건 포함
