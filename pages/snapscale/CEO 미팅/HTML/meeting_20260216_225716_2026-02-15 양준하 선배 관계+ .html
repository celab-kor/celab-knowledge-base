투자 라운드 절차 & 지분 구조 처리 전략
▼

논의 내용

  • 텀싯(Term Sheet) 수령 직후, 내부 합의 → 주주간 계약서 사인 → 텀싯 사인 순서가 법적으로 가능한지 변호사 자문 필요
  • 30% 지분 확정 사실을 투자자에게 공표하되, 아직 사인하지 않은 사실은 명시하지 않을 계획
  • 투자자 반응을 보면서 선택지를 유연하게 넓혀두는 전략 채택 (투자자가 이유를 들어 어렵다면 그때 추가 협상 진행)
  • 텀싯 수령 전까지 투자자들에게 "30% 지분 확정" 메시지를 우선 전달하여 투자 분위기를 조성

Insight

  • ⚡ 전략적 유연성
    사인 전 지분 확정 공표는 투자 협상력을 높이는 동시에 리스크 헤징 수단으로 작용. 단, 법적 검토 없이 진행 시 신뢰 문제로 이어질 수 있어 변호사 자문이 선결 과제.
  • ⚖ 법적 리스크
    주주간 계약서와 텀싯의 서명 순서는 법적 우선권 및 분쟁 발생 시 해석에 영향. 실행 전 법무 검토를 완료해야 사인 시퀀스를 확정할 수 있음.
  • 📌 투자자 커뮤니케이션
    투자자 대상 메시지를 선제적으로 통일하여 혼선 방지. "30% 확정 + 방향성 설명" 프레임은 투자자 심리적 확신 제고에 효과적.

배당가능이익 구조 & 스톡옵션 설계 검토
▼

논의 내용

  • 배당가능이익 = 순자산 − 자본금 − 법정준비금 − 이익준비금 − 미실현이익
  • 현재 자본금 100만 원 유지, 법정준비금·이익준비금·미실현이익은 사실상 0원에 근접
  • 향후 POC·정식 계약 매출 발생 시 인건비·운영비 차감 후 순이익 대부분이 배당가능이익으로 전환되는 구조
  • 순이익 수억 원 수준에서 자사주 매입을 통한 스토킹옵션(Stalking Option) 부여 가능성 추산 — 변호사 추가 확인 필요

Insight

  • 💰 보상 설계 관점
    초기 자본금이 최소화된 구조는 배당가능이익 풀을 극대화하는 데 유리. 매출 발생 즉시 스톡옵션 재원 마련이 가능하여 핵심 인재 보상 설계에 강점.
  • 🔍 법적 확인 필수
    자사주 취득 기반 스토킹옵션 부여는 상법상 요건(배당가능이익 범위 내 자사주 취득)을 충족해야 하므로 실행 전 변호사 확인이 선결 조건.
  • 🔗 영입 협상 연결
    양준하 선배 등 4~5년 후 Exit을 원하는 인재에게는 지분보다 스톡옵션이 구조적으로 합리적. 이 논의가 영입 조건 설계의 근거로 직결됨.
3
양준하 선배 영입 전략 — 컨텍스트 & 협상 방향
▼

배경 및 현황

  • 학교 선배·멘토 멘티 관계 (공식 멘토링은 2025년 종료, 이후에도 꾸준히 교류)
  • 삶의 가치관·인터넷 문화·음악 취향 등에서 강한 개인적 유대감(본딩) 형성
  • 코파운더 가능성을 비공식으로 타진 — "지분풀이 남아있냐" 수준의 탐색 단계
  • 하이퍼리즘 CTO 출신 — LLM 효율화 케이스 스터디 다수 보유, 바이브 코딩 트렌드에 정통
  • 선제 컨택 이유: "물리 세계 + AI" 방향성에 공감, 에너지·제조업에 AI를 먼저 접목하는 전략에 설득력 느낌

선배 측 요구 조건 & 우려사항

  • Exit 가능성: 4~5년 후 Exit 가능 여부 확인 → 스톡옵션 구조가 적합
  • 개발 몰입 환경: 주 단위로 아이템이 뒤바뀌는 환경은 어려움 표명
  • 제조업 도메인 이해 부족: 마켓캡, 진입 가능성, 시니어 퍼스널리티에 대한 정보 부족 → 2/22 미팅에서 상세 설명 예정
  • 기업가치 기대: 4~5년 내 수천억 원 기업가치 가능 여부 확인 필요

2인 vs 3인 체제 논의

  • 현재 대표(상윤) + 박 부장(이중윤) 2인 체제로 합의된 상태
  • 3인 체제 전환 조건: "3인 체제일 때 최종 시장 파이가 2인 체제보다 명확히 커진다"는 근거가 선배 측에서 제시되어야 함
  • 지분 부여(코파운더)보다는 스톡옵션 2~3% + 연봉 3,000~4,000만 원 수준이 현실적으로 검토됨
  • 박 부장: "아직 키워드 언급 수준, 옵션화만 인지" — 2/22 미팅 후 재논의 예정

스냅스케일의 어필 포인트

  • 물리 세계(제조·에너지) × AI의 교차점을 선제적으로 공략하는 희귀한 팀
  • 랜딩페이지 없이도 다수 기업 문의 인바운드 발생 — CLG 폭발력 입증
  • 보안·백엔드 위임 + CTO 역할 부여로 기술 리더십 발휘 공간 확보
  • 미국 팀보다 스냅스케일에 더 강한 관심 표명한 정황 활용

Insight

  • 🤝 협상 포지셔닝
    현 단계는 "스냅스케일이 어필하는" 구도이나, 희귀한 팀·독보적 도메인·선제 시장 진입을 강조하면 상호 어필 구도로 전환 가능. 압박 협상보다 파트너십 프레이밍이 유효.
  • 📊 보상 구조 명확화
    스톡옵션 2~3% + 연봉 협의 방향은 Exit 지향 인재에게 적합하나, 하이퍼리즘 재직 연봉 수준 파악 후 조정 필요. 연봉이 너무 낮을 경우 스톡옵션 비율 상향 협상 여지 검토.
  • 🧠 도메인 설득 전략
    제조업에 대한 정보 공백이 가장 큰 장애물. 2/22 미팅에서 밸류체인 구조·마켓캡·CLG 파이프라인을 시각화하여 전달하면 의사결정 가속 가능.
  • ⚠ 3인 체제 리스크
    3인 체제 전환은 의사결정 구조 복잡도 증가와 지분 희석을 수반. 선배가 명확한 시장 확장 근거를 제시하지 못할 경우 2인 체제 유지가 합리적.

제조업 시장 분석 — 마켓캡 & 진입 가능성 자료 준비
▼

제조업(플랜트) 밸류체인 구조

  • 발주처 → 라이선스사 → EPC사 → 협력업체 → 설계회사·시공사 (심플 트리 구조)
  • IT 업계 대비 레이어 수 적고 위계 구조 명확 — 의사결정 경로 예측 용이
  • 설계 영역 시니어: 점잖고 전문성 높음 + 소프트 군대 문화 혼재, 순수한 내면에 터프한 말투

마켓캡 추산 (Top-down & Bottom-up)

  • Top-down: 1조 원 프로젝트 기준 인건비 5~8%, 낭비 금액 ≈ 100억/프로젝트 × 3건 = 300억/대기업. 국내 대기업 3곳 × 300억 = 연 900억 원
  • Bottom-up: 프로젝트당 설계 인원 200명 × 평균 연봉 7,000만 원 = 140억 → 낭비 금액 100억과 근사 일치
  • 한국 시장 (중견·중소 포함): 약 1,500억 원
  • 해외 시장 (국내 50배 이상 규모): 약 7.5조 원 이상
  • 4~5년 내 시나리오: 국내 대기업 2곳 + 해외 대기업 1곳 + 국내외 중견 20곳 → 연매출 1,000억 원 수준 가능
  • 현실적 유의사항: 초기 매출은 특정 부서(공정·배관) 한정으로 수백억 이하 예상, 구독제 전환이 핵심 병목 (마케나락스 사례: 연구독 99억 수준도 어려움)

진입 가능성

  • 1개 기업에서 "정확도 + 속도 보장" 달성 시 업계 내 소문 확산 → 인바운드 폭발적 증가 예상
  • 현재 CLG 파이프라인 (랜딩페이지 없이 문의 인바운드): 케이조(KJO·3월 구축 계약 예정), 한홀·필지(잔사 대보 예정), 삼두·이수(견적서 송부), 현앤·현중(사내 데이터 테스트 예정), 화학물질안전(세미나 예정)
  • 현중 제외 전 건이 자연 인바운드 — 레퍼런스 1건 확보 시 CLG 폭발 시나리오 현실적

Insight

  • 🎯 고객 설득 전략 보강 필요
    CEO 오케이 후에도 실무자(부장급)의 저항이 의사결정 지연의 핵심 원인. "99% 정확도" 강조보다 "왜 구현 가능한가"의 기술 근거와 "실무자 역할 보장" 메시지가 더 효과적.
  • 🔄 엔지니어 대체 → 리뷰어 전환 프레임
    현재 "엔지니어 대체" 어조가 실무자 거부감 유발. "반복 노동 제거 → 엔지니어가 리뷰어·컨펌자·확장자로 진화"하는 프레이밍으로 전환 시 심리적 저항 완화 가능.
  • 📈 구독 전환 모델이 병목
    마케나락스 사례처럼 연구독 전환은 시간과 CLG 역량을 매우 많이 요구. 초기 프로젝트 단위 과금 → 단계별 구독 전환 로드맵 수립이 필요.

개발팀 구성 전략 — 슈퍼스타 팀 vs 중소기업형 팀
▼

팀 구성 방향

  • 슈퍼스타 팀 우선 지향: 컴공 베이스 + 데이터 구조 이해 + 바이브 코딩 가설 설정 능력 보유자 중심
  • 슈퍼스타 5인 구성안: 대표(상윤) + 재윤 + 양준하 선배 + 신규 채용 2인
  • 현 팀원 평가: 재윤 → 슈퍼스타급 확실, 태윤 → 데이터 구조 이해 부족으로 속도 저하 (재계약 재고), 다손 → 리모트로 평가 어려움 (2~3개월 계약직으로 검증 예정)
  • 차선 옵션: 슈퍼스타까지 아니더라도 "평균 이상 똑똑한 개발자" 7~8인 규모

양준하 선배 역할 설계 (가안)

  • 보안·백엔드 파트 위임 + CTO 역할
  • 개발자 리크루팅: 현 단계에서는 대표가 주도 (도메인 이해 기반 매력 어필이 핵심 — 선배 적응 후 단계적 위임)
  • 필수 선결 조건: 실제 로우 레벨 작업(End-to-end 태스크) 직접 수행 후 팀원 역할 분배 결정

비개발 인력 구성 (박 부장 영역)

  • 박 부장(이중윤) + 대표 + 도메인 주니어 엔지니어 1인 (대리급)으로 운영 가능
  • 시니어 도메인 전문가는 초기 레퍼런스 구축 이후 선별적 영입

Insight

  • 💡 슈퍼스타 팀 조건
    컴공 + 도메인 학습 능력 + 바이브 코딩 피드백 루프 능력의 조합이 핵심. 개발 속도의 격차는 이 세 조건의 유무에서 결정되며, 이를 검증하는 2~3개월 계약직 검증 트랙이 합리적.
  • 🏗 CTO 온보딩 리스크
    CTO 역할 부여 전 실제 작업 참여 조건은 현실적이나, 선배의 기대치와 부여 가능한 역할 범위를 사전에 명확히 정렬하지 않으면 입사 후 갈등 발생 가능. 역할 정의서 사전 작성 권장.
  • 🔁 리크루팅 전략
    현재 대표가 직접 리크루팅하는 방식(도메인 이해 기반 매력 어필)이 핵심 경쟁력. 이 어필 역량이 이전되기 전 CTO에게 채용 권한 위임은 시기상조.

투자 라운드 타이밍 & CLG 파이프라인 가속 전략

문제 진단

  • 현재 발표 메시지: "이 모든 것이 가능하다 (99% 정확도)" → 실무자에게 "AI가 우리를 대체한다"로 받아들여짐
  • CEO 승인 후에도 부장급 실무자가 의사결정을 뒤로 미루는 패턴 반복 확인
  • "제품 완성 후 보겠다" 응답 증가 → 선 체험 없이 계약하기 어려운 인식의 장벽

메시지 전환 방향 (브레인스토밍 필요)

  • 기존 프레임: "AI가 엔지니어를 대체한다" → 신규 프레임: "엔지니어가 실무자에서 리뷰어·컨펌자·확장자로 진화한다"
  • 핵심 포인트: 단순 반복 노동(로그 데이터 검색, 문서 정리 등)만 제거 → 진짜 엔지니어링에만 몰입 가능
  • 인력 감축이 아닌 "동일 인원으로 더 많은 프로젝트 수주·수행 가능" → 매출 확장 도구로 포지셔닝
  • 산업의 지식을 깊이 이해하고 고민한 팀임을 사람 대 사람으로 느끼게 해야 함
  • 심리적 단계: 부정 → 분노 → 시니컬 → 수긍 → 안주 → 절망 각 단계에 대응하는 메시지 준비 필요

Insight

  • 🧩 심리 설계 필요
    기술 발표에서 감정 관리로 축이 이동해야 함. 실무자의 심리적 저항 단계를 예측하고 각 단계별 대응 스크립트를 준비하는 것이 영업 전환율 제고의 핵심.
  • 🎭 브랜드 포지셔닝
    "AI 스타트업이 돈 버는 것"이 아닌 "이 산업을 진정으로 이해한 팀"으로 포지셔닝 전환. 퍼스널리티 공감 + 산업 공감이 기술 신뢰 형성보다 선행되어야 함.
  • 🔧 실행 방법
    AI 브레인스토밍 세션 (별도 일정)을 통해 최적 메시지 도출 후 공식 발표 자료에 반영. 이를 별도 태스크로 지정하고 기한 설정 권장.
