🏭 필즈엔지니어링 국가과제 현황

산업부 주관 국가 연구개발 과제 수주 완료. 2026년 3월 말 연구비 입금 확인. 2차 년도(2026년) 수행 중. 과제는 총 3개 세부 과제로 구성: ① LNGC 재기화 설비(750 MMSCFD), ② 소형 재기화 설비(20 MMSCFD), ③ CO₂ 해저 주입 시설.

🎯 회의 핵심 목적

국가과제 수행 딜리버러블을 분석하여 AutoFlow(SnapScale)가 각 단계별로 개입할 수 있는 범위를 확정하고, 필즈 측에 요청할 입력 데이터 목록 및 견적서를 다음 주 화요일까지 발송하기 위한 논의.

📌 SnapScale 관점에서의 기회

이번 국가과제는 테스트베드(실증 사업)로 최적. 국가과제 특성상 결과 부분 허용, 리스크 낮음. 성공 레퍼런스 확보 시 Series A/B 라운드에 강력한 PoC 근거 제공 가능.

⚙️ 핵심 기술 배경

LNGC → FSRU(재기화 유닛) 전환은 박규호 부장이 기본 설계 시절부터 연구해온 분야. 동남아 중심으로 실제 수주 사례 다수. SnapScale 온톨로지 기반 AI는 이 공정 구조를 그래프로 모델링하여 자동화 가능.

LNGC 재기화(FSRU) 공정 기술 구조 이해
▼
박규호 부장: "탱크에 극저온 LNG가 마이너스 163도에 있다. 그걸 목적하는 데까지 보낼 거다. 그 설계기술서가 나올 거고요. 장비 두 라인이 있는데 — BOG 자동기화 라인과 펌프 강제기화 라인 — 이 두 가지 합산이 리퀴어먼트를 만족해야 합니다."

📦 재기화 공정 핵심 구조

① BOG(Boil-Off Gas) 라인: 탱크 내 자동 기화 가스 → 압축 → 요구 조건 송출

② 강제기화 라인: LNG 펌프로 액체 추출 → 열교환기(Glycol Water) → 기화 → 압축 → 요구 조건 송출

최종 요구조건: 750 MMSCFD / 목표 압력·온도 (연구계획서 기재)

⚙️ 하이시스 모델링 핵심 과제

피드 정의: LNG 성분(몰분율: 메탄 ~95%, 에탄 등) + BOR(Boil-Off Rate) 산정

목적값 찾기(Adjust): 열교환기 후단 온도·압력이 요구조건에 수렴하도록 반복 최적화 — 핀치 분석(Pinch Analysis) 필요

장비 사이징 확정 → Equipment List 출력

💡 안건 인사이트
  • 피드·프로덕트 정의만 있으면 AutoFlow 시작 가능: 공정 구조 자체는 기존 논문·특허로 완벽히 정의됨. AI가 논문 서베이 → 장비 리스트 → 하이시스 자동 드로잉을 순차 실행 가능.
  • Adjust 함수 자동화가 핵심 기술 과제: 반복 최적화(이터레이션)를 AI 코딩으로 자동화하면 석사 논문 1편 분량의 작업을 수 시간 내 완료.
  • 현대중공업 LFSS 사례 직접 재활용 가능: 박 부장이 보유한 설계 표준이 현재 과제와 동일한 코드·구조를 따름. 온톨로지 구축에 즉시 활용 가능 (외부 유출 금지).

▶ 단계별 AutoFlow 개입 범위

단계 필즈 수행 내용 AutoFlow 역할 필즈 제공 데이터 적용 난이도
① Design Philosophy 논문·특허 조사, 설계 기준 수립 논문 자동 서베이, 장비 후보 리스트 도출, 설계 기준 문서 초안 생성 산업표준 코드(IGC, 선급 규정), 750 MMSCFD 요구 조건 즉시 가능
② Basis of Design Feed 성분, Product 요구조건 확정 기준 템플릿 자동 생성, 논문 근거 자동 참조 LNG 몰분율, 목표 압력·온도·유량 즉시 가능
③ PFD + 하이시스 시뮬레이션 하이시스 시뮬레이션 수행 Feed+Product 조건 입력 → 하이시스 자동 드로잉 → Adjust(목적값 찾기) 자동화 Feed 조건, Product 요구조건 (BOD에서 도출) 2~3개월 개발
④ H&MB 자동 추출 하이시스에서 H&MB 엑셀 추출 워크북 자동 추출, 라인별 온도·압력·유량 자동 정리 하이시스 파일 개발 필요
⑤ P&ID 생성 (3차 년도) P&ID 도면 작성 PFD → P&ID 변환(Safety Logic 포함) — 현재 Phase 3 계획 단계 PFD, Safety Logic 기준 장기 개발
⑥ Equipment/Instrument List 장비·계기 데이터 시트 작성 P&ID 입력 → Equipment List, Instrument List, Data Sheet 자동 생성 (현재 구축 완료) P&ID 도면 즉시 가능
⑦ Line List + Hydraulic Calc 라인 리스트 및 수리 계산 H&MB → 유속 기준 자동 라인 사이즈 계산, 라인 리스트 자동 생성 속도·압력 드랍 기준 즉시 가능
⑧ MTO + 견적 물량 산출 및 구매 견적 전 단계 데이터 정합성 검증 → MTO 자동 집계, PO 문서 자동 생성 벤더 데이터 즉시 가능
⑨ Commissioning Procedure 시운전 시나리오 작성 Operation Scenario 별 밸브 시퀀스 자동 문서화 확정된 P&ID, 운전 시나리오 개발 필요
⚠️ 핵심 경계 — 라이선서 역할 침범 금지
  • AI가 개념 설계를 "보장"하거나 완성물로 제공하는 것은 사업 리스크: "필즈가 수주한 연구 용역을 서브콘으로 우리가 받아버리는 상황이 될 수 있다"(박규호)
  • AutoFlow의 역할: 정확도 보장이 아닌 "인간이 실제 일을 하는 프로세스를 더 단축"하는 속도 도구
  • 초안의 정확도보다 작업 시간 절감 측정(KPI)이 핵심 성과 지표
💡 안건 인사이트
  • ① → ④ 구간이 즉시 착수 가능한 영역: 기존 온톨로지 기반 구조 + 박 부장 도메인 지식 조합으로 3~4개월 내 구현 가능성 확인.

  • ⑥~⑧ 구간은 SnapScale 현재 강점: P&ID 이후 디테일 엔지니어링 자동화는 현재 구축된 AutoFlow의 핵심. 이 구간에서 최대 ROI 창출 가능.

견적 규모 및 구성 방향
▼
