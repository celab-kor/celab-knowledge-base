안건 1. 온톨로지 구축 및 데이터 소유권 논의

온톨로지의 산업 표준화 가능성 ▼
대림 직원
"온톨로지가 공유적이어야 한다는 점에서, 특정 개인이 아니라 A, B, C 산업계 전체의 합의가 필요할 것 같습니다. 스냅스케일이 작년 8월부터 시작했다고 하면 6개월이라는 짧은 시간인데, 이 기간에 산업계 합의가 이루어졌는지 궁금합니다."
김상윤 대표
"정확히는 작년 10월부터 본격적으로 도메인 이해를 시작했습니다. 저희가 구축한 온톨로지는 특정 기업에 국한된 것이 아니라, 플랜트를 짓는다면 응당 들어가야 되는 기본 개념들로 구성되어 있습니다. 예를 들어 하나의 Vessel이라도 A기업과 B기업이 다른 심볼 레전드를 사용할 수 있지만, 기본 개념은 동일합니다. 따라서 200~300% 커스터마이징이 필요한 것이 아니라, 합의된 개념 하에서 심볼이나 산업 시스템 등 국지적인 커스터마이징만 필요합니다."

핵심 인사이트

스냅스케일의 온톨로지는 산업 표준 개념을 기반으로 구축되어 있어, 기업별 완전한 재구축이 아닌 부분적 커스터마이징만 필요
6개월의 개발 기간은 실제로는 도메인 스터디를 포함한 기간이며, 핵심 개발은 더 오랜 준비 기간을 거침
온톨로지의 표준화는 산업계 전체 합의보다는 플랜트 엔지니어링의 보편적 개념을 구조화하는 것에 초점
데이터 소유권 및 지식재산권 문제 ▼
대림 직원
"회사가 만든 온톨로지에 대한 지식재산권은 어떻게 되나요? 회사만의 온톨로지를 만들어주신다면 그 소유권은 어떻게 되는지 궁금합니다."
김상윤 대표
"플랜트 엔지니어링 기업 내부에서 활용하신 딜리버러블 데이터의 소유권이 저희에게 있으면 당연히 안됩니다. 다만 저희가 제공하는 온톨로지와, 그것을 기반으로 쌓인 엔지니어링 구조, AI 파라미터 값 등에 대한 소유권은 저희가 가져가야 합니다. 중요한 것은 기업의 데이터 소유권은 그대로 유지하되, AI를 활용하면서 만들어낸 부가가치 데이터의 소유권을 어떻게 설정할지 같이 고민이 필요합니다."

합의 사항

• 기업 원천 데이터: 플랜트 엔지니어링 회사가 완전 소유
• 온톨로지 구조: 스냅스케일이 제공하며 라이선스 형태로 사용권 제공
• AI 생성 데이터: 향후 협의 필요 (온톨로지 업데이트, 파라미터 값 등)
• 결론: 상호 이익을 고려한 데이터 소유권 구조 필요, 세부 계약 조건은 POC 이후 논의

김상윤 대표
"저희 온톨로지 구조는 파이프 하나하나가 물리적 좌표까지의 값을 가지고 있습니다. 3D 모델링이 되려면 각각의 객체들이 좌표 값, 물리적 사이즈, 길이 등을 가져야 하는데, 이런 것들을 온톨로지에 그대로 반영하고 있습니다. 따라서 LLM이 정확하게 에이전틱한 액션을 수행할 수 있습니다."

김상윤 대표
"과거 데이터베이스를 기반으로 판정합니다. 회사에서 그동안 수행했던 과거 프로젝트의 Instrument List 작성 기준이 그대로 녹아들어 있기 때문에, AI가 과거 파일을 참조해서 리스트를 작성합니다. 과거 데이터를 학습해서 비슷하게 만들어준다고 보시면 됩니다."
대림 직원
"현재 보면 A, B, C 세 사람이 프로젝트를 하면 형식이 다 다를 수 있는데, 이런 자료가 올라가면 AI가 어떻게 판단할까요?"
김상윤 대표
"중요한 부분은 하나의 일관된 기준으로 쓰느냐입니다. 프로젝트마다 스타일은 다를 수 있어도 하나의 프로젝트 안에서는 일관된 철학을 가지고 작성하기 때문에, 그것을 보장하는 게 중요합니다. 베스트 프랙티스를 정확하게 모사하는 방식으로 리스트를 작성합니다."
박규호 부장
"ACC라는 견적 툴의 볼류메트릭 최적화 모델처럼, 컬럼 전후단에 붙어야 하는 장치류를 하나의 세트로 갖고 있습니다. 스탠다드 타입과 SIS(Safety Instrument System) 강화 모델 등 여러 포맷이 있고, 사용자가 안전 마진을 어떻게 설정하느냐에 따라 펌프 전후단 밸브 구성이 달라집니다."

템플릿 기반 접근으로 베스트 프랙티스 적용 및 신속한 스타일 전환 가능

안건 4. 온톨로지 정확도 검증 및 오류 처리

정확도 평가 방법론 ▼
대림 직원
"온톨로지를 구축했을 때 정확도를 어떻게 평가하나요? 95% 이상만 필터링하고 미만은 하이라이트를 시켜주는지, 아니면 해외 저렴한 곳에서 보완 작업을 하는지 궁금합니다."
김상윤 대표
"중요한 점은 결국 결과가 정확하냐에 초점을 맞춥니다. 온톨로지 기반으로 만든 라인 리스트가 실제 완성 버전과 비교했을 때 얼마나 정확한가를 판단합니다. 첫 번째는 여러 딜리버러블을 검증했을 때 결과가 정확하게 나오는지 확인하고, 두 번째는 사람이 수동으로 온톨로지 구조를 검사합니다. 외부 자문단에게 예외 케이스나 문제점을 검토받아 정확도를 높이고 있습니다."
대림 직원
"정확도가 어느 정도인가요?"
김상윤 대표
"딜리버러블 기준으로 검사했을 때 약 98% 정도 나오고 있습니다. 다만 이것도 개념증명 단계에서의 퍼센티지이기 때문에, 실제 기업과 협력 개발로 들어갈 때는 추가 개발이 필요할 수 있습니다."

정확도 검증 체계

1차 검증: 딜리버러블 비교를 통한 결과 정확도 측정 (현재 98%)
2차 검증: 도메인 전문가의 수동 온톨로지 구조 검사
3차 검증: 외부 자문단을 통한 예외 케이스 검토
한계점: 현재는 개념증명 수준이며, 실제 도입 시 추가 정교화 필요

안건 7. 3D 모델링 데이터베이스 연동

S3D/E3D 데이터베이스 활용 ▼
대림 직원
"S3D나 E3D를 많이 쓰는데, 모델 데이터베이스(DB)에 대한 접근도 가능한가요? E3D를 통하면 정보가 DB에 저장되고 여러 협력사 모델러들이 동시에 작업하는데, 3D 뷰에 대한 DB 접근이 가능하면 활용도가 높아질 것 같습니다."
김상윤 대표
"제약사항은 많지만 구현은 가능한 부분입니다. 스틸 스트럭처나 재질, 파이프랙 같은 기본 정보를 주면 강도 계산이 가능하고, 최적 루트로 최저 물량이 나오게 오토 루팅을 해주는 것도 가능합니다. 다만 데이터 트랜스퍼링 작업이 필요합니다."
박규호 부장
"배관 설계는 작업량이 제일 많습니다. 리스트 작업, 자재 관리, 모델링 상세 설계 등이 복잡하고, 인풋이 바뀌는 것에 대한 리비전이 많아서 AI 기반 설계 변화를 시도할 여지가 많습니다. 과거 학습 데이터를 활용해서 어시스트 개념으로 최적화율과 정확도를 높이면 반자동 느낌으로 활용 가능할 것 같습니다."

핵심 인사이트

3D 모델링 DB 접근은 기술적으로 가능하나 현재는 공정 설계 중심으로 개발됨
배관 설계가 가장 많은 작업량과 리비전을 요구하므로 AI 자동화 효과가 클 것으로 예상
완전 자동화보다는 과거 데이터 기반 어시스트 개념이 현실적
3D 확장을 위해서는 배관 설계 엔지니어와의 긴밀한 협업 필요
