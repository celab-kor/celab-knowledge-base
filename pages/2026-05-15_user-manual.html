📖 지식창고 사용 매뉴얼

누구나 따라 할 수 있는 회의록·문서 발행 가이드
2026-05-15 · 박규호 부장 워크플로 기준
① 한눈에 보기 ② 회의 영상부터 시작 ③ 외부 HTML 발행 ④ Claude에게 부탁 ⑤ 발행 확인 ⑥ 자주 묻는 질문 ⑦ 문제 해결
이 매뉴얼은 한 줄로 요약하면:
"회의록 만들고 → 발행하기 버튼 한 번 → 끝. 카테고리 폴더로 옮기지 마세요."

⌨️ 단축키 한 줄 요약

어디서든 ⌥ ⌘ K (Option + Command + K) → Chrome에서 지식창고 갤러리로 즉시 점프

부팅 시 자동 실행도 활성화됨 → 잠금 해제하면 Chrome에 갤러리가 이미 떠 있음.

2시나리오 A — 회의 영상부터 시작

🎬

출발점: 회의 영상 또는 오디오 파일 (mp4, mp3, m4a 등)

예: 줌 녹화본, 클립아이 녹음, 아이폰 음성메모

flowchart LR M[🎥 회의 영상] --> N[Rev.3 앱 실행] N --> O[영상 끌어다 놓기] O --> P[자동 변환 대기
5~30분] P --> Q[HTML 자동 생성] Q --> R[앱 안에서 편집] R --> S[발행하기 버튼] S --> T[✅ 완료] style M fill:#63b3ed,color:#0d1b2a style S fill:#c9a84c,color:#0d1b2a style T fill:#34d399,color:#0d1b2a

📋 따라 하기

1

영상회의록 추출기 Rev.3 실행

데스크탑(또는 앱 폴더) 안의 영상회의록추출기_Rev3_스마트실행.command을 더블클릭합니다.

처음 실행 시 "확인되지 않은 개발자" 경고가 나오면, 파일 우클릭 → "열기" 선택.

2

영상 또는 오디오 파일 끌어다 놓기

앱 창 안에 회의 영상을 드래그합니다.

출력 폴더는 처음 한 번만 지정해두면 다음부터 기억합니다.

3

자동 변환 대기 (5~30분)

이 시간 동안 자동으로 진행되는 일:

  • 영상 → 오디오 추출
  • 오디오 → Whisper로 텍스트 변환
  • 텍스트 → Claude API로 HTML 회의록 생성

변환 중에는 다른 일 하셔도 됩니다. 알림으로 완료를 알려줍니다.

4

앱 안에서 HTML 검토·편집

"보기" 버튼을 누르면 생성된 HTML이 앱 안 미니 브라우저로 열립니다.

내용 수정이 필요하면 "편집" 버튼으로 텍스트 편집창 진입.

5

"발행하기" 버튼 클릭

발행 다이얼로그가 뜹니다. 다음 5가지를 입력하세요:

항목예시
제목2026-05-15 김상윤 대표 미팅 — Q2 전략
설명Q2 ARR 목표 및 가격정책 협의 결과 정리
카테고리CEO미팅 / 팀미팅 / 투자 / IR / 전략 등
태그가격정책, Q2, OKR, ARR
이모지🎯

입력 후 "발행하기" → 자동으로 깃 푸시 + 슬랙 알림까지 완료.

✅ 끝났습니다 더 이상 할 일이 없습니다. 30초 후 갤러리에 카드로 자동 노출됩니다.

3시나리오 B — 이미 HTML이 있는 경우

📁

출발점: 어딘가에서 받은 HTML 파일

예: 다른 사람이 만들어준 회의록, 노션에서 추출한 HTML, 옛날 파일 재발행

flowchart LR H[📄 HTML 파일] --> I[데스크탑의
지식창고_발행대기
폴더로 드래그] I --> J[발행대기처리.command
더블클릭] J --> K[자동 처리] K --> L[✅ 완료] style H fill:#63b3ed,color:#0d1b2a style J fill:#c9a84c,color:#0d1b2a style L fill:#34d399,color:#0d1b2a

📋 따라 하기

1

HTML 파일을 인박스로 드래그

데스크탑의 📁 지식창고_발행대기 폴더 안으로 끌어다 놓습니다.

여러 파일을 한 번에 넣어도 됩니다.

2

같은 폴더 안의 ⚡발행대기처리.command 더블클릭

터미널 창이 떴다가 자동으로 처리합니다:

  • 파일명을 URL-안전한 형식으로 변환
  • 날짜 prefix 자동 추가 (없는 경우)
  • 지식창고 pages/ 폴더로 이동
  • 깃 푸시 + 슬랙 알림

처리가 끝나면 macOS 알림으로 "○개 페이지 발행 완료" 표시.

⚠️ 주의 이 경로는 갤러리 인덱스(index.html)에는 자동 등재되지 않습니다. 슬랙 알림은 가지만, 갤러리 카드로 노출되려면 별도로 메타데이터(제목·카테고리·태그)를 추가해야 합니다. 가능하면 시나리오 C (Claude에게 부탁)을 사용하세요.

4시나리오 C — Claude에게 부탁하기

💬

출발점: 머릿속의 내용 또는 텍스트 메모

예: 회의에서 든 생각, 카톡 대화 정리, 보고서 초안

flowchart LR A[💬 Claude Code에 요청] --> B[Claude가 HTML 작성] B --> C[갤러리 인덱스 자동 등재] C --> D[자동 푸시 + 슬랙] D --> E[✅ 완료] style A fill:#a78bfa,color:#0d1b2a style E fill:#34d399,color:#0d1b2a

📋 따라 하기

1

Claude Code 실행

터미널에서 claude 입력 또는 cc alias 사용.

지식창고 폴더로 자동 진입: cd ~/Documents/celab-knowledge-base

2

자연어로 요청

예시 프롬프트:

  • "오늘 김상윤 대표와 나눈 가격정책 대화를 정리해서 지식창고에 발행해줘"
  • "투자사 미팅 준비용 1-페이지 브리핑 만들어서 카테고리는 IR로 발행"
  • "이 텍스트 내용을 HTML로 정리해서 지식창고에 올려줘: [텍스트]"
3

Claude가 자동으로 모두 처리

Claude가 다음을 한 번에 처리:

  • pages/ 폴더에 HTML 생성
  • index.html 갤러리에 카드 등재
  • 깃 푸시
  • 슬랙 알림 발송

마지막에 공개 URL과 슬랙 메시지 발송 결과를 보여줍니다.

5발행이 잘 됐는지 확인하는 3가지 방법

1

슬랙 채널 확인

발행 후 약 5초 이내에 슬랙으로 링크가 도착합니다.

📝 제목 · 설명 · 태그 · 카테고리 · 날짜가 모두 표시됩니다.

2

지식창고 갤러리 방문

celab-kor.github.io/celab-knowledge-base

맨 위에 새 카드가 보이면 성공. 약 30초~1분 지연 가능 (GitHub Pages 배포 시간).

3

슬랙 링크 클릭 → 발행된 페이지 직접 확인

가장 확실한 방법. 링크가 열리고 본인의 HTML 내용이 정상적으로 보이면 끝.

6자주 묻는 질문

Q1. 발행한 페이지를 수정하고 싶어요

Claude Code에서 "지식창고의 2026-05-15_XXX.html 페이지에서 ○○ 부분을 △△로 수정해줘"라고 요청하세요. 수정 후 자동으로 깃 푸시·슬랙 알림까지 처리됩니다.

Q2. 발행한 페이지를 삭제하고 싶어요

Claude Code에서 "2026-05-15_XXX.html 페이지를 지식창고에서 제거해줘"라고 요청하세요. pages/ 폴더에서 파일 삭제 + index.html 갤러리에서 카드 제거를 함께 처리합니다.

Q3. 카테고리를 새로 만들고 싶어요

현재 카테고리: 투자 · IR · 분석 · 전략 · 기타 · 교육. 새 카테고리(예: "회의록")를 추가하려면 Claude Code에 요청. index.html에 CSS 색상 정의 + 필터 버튼 추가를 함께 처리합니다.

Q4. 비공개로 발행하고 싶어요 (슬랙 알림 없이)

현재 슬랙 알림은 자동입니다. 비공개가 필요하면 pages/_drafts/ 폴더(gitignored)를 사용하시면 됩니다. 단, 깃허브에 올라가지 않으므로 본인 Mac에서만 볼 수 있습니다. 사용 의향이 있으시면 알려주세요.

Q5. 슬랙 채널이 다른 곳이 됐어요. 어떻게 바꾸나요?

슬랙 webhook URL을 변경: ~/Documents/celab-knowledge-base/scripts/.env.local 파일에서 CELAB_SLACK_WEBHOOK 값을 새 URL로 교체. 영상회의록추출기 Rev.3 앱의 webhook은 별도로 main.js에 하드코딩되어 있어 거기도 같이 수정 필요.

Q6. 자동 동기화는 언제 일어나나요?

"자동동기화.command" 또는 LaunchAgent가 일정 간격(설정에 따라 다름)으로 sync.sh를 실행합니다. 즉시 반영이 필요하면 Claude Code에 "지금 푸시해줘" 또는 직접 bash sync.sh.

Q7. 기존 02 CELab/인터뷰/{카테고리} 폴더는 어떻게 하나요?

그대로 두세요. 새 회의록만 새 워크플로우로 처리. 기존 파일은 점진적으로 인박스(시나리오 B)나 Claude(시나리오 C)로 마이그레이션. 한 번에 일괄 이동은 위험합니다.

7문제 해결

!

"확인되지 않은 개발자" 경고가 떠요

macOS Gatekeeper 경고. 파일 우클릭 → "열기" → 한 번 더 "열기" 확인. 이후엔 더블클릭으로 정상 실행.

!

발행했는데 슬랙에 메시지가 안 와요

1. 깃 푸시는 성공했는지 확인: cat ~/Documents/celab-knowledge-base/.sync.log | tail -20

2. webhook URL이 유효한지 확인 (Claude에 "슬랙 webhook 테스트해줘" 요청)

3. 슬랙 채널 권한 확인 (해당 webhook이 가리키는 채널에 본인이 속해 있는지)

!

갤러리에 카드가 안 보여요

3가지 원인 중 하나:

  • ① index.html의 PAGES 배열에 등재가 안 됨 → Claude에 "이 페이지를 갤러리에 등재해줘" 요청
  • ② GitHub Pages 배포 지연 (약 1분 대기 후 새로고침)
  • ③ 브라우저 캐시 → 강제 새로고침 (Cmd+Shift+R)
!

발행대기처리.command가 실행되다가 실패해요

로그 확인: cat ~/Documents/celab-knowledge-base/.sync.log | tail -30

주요 원인: 깃허브 인증 만료(GitHub Token 갱신 필요) 또는 디스크 공간 부족.
