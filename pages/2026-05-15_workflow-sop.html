3해야 할 일 / 하지 말아야 할 일

✅ 해야 할 일

  • 앱 발행 버튼 사용 — pages/·index.html·깃·슬랙 다 자동 필수
  • 카테고리·태그 메타 입력 — 갤러리 필터에서 분류 기준 필수
  • 파일명 prefix YYYY-MM-DD_제목 — 시간순 자동 정렬 권장

❌ 하지 말아야 할 일

  • 11개 카테고리 폴더로 수동 이동 폐기 — 메타로 분류
  • pages/ 안에 하위 폴더 생성 금지 — 슬랙 알림 안 감, 갤러리 등재 안 됨
  • 수동 git push 불필요 — 앱이 함
  • 수동 슬랙 메시지 불필요 — 앱이 함

4기존 11개 분류 폴더는 어떻게?

현재 ~/Documents/02 CELab/00. 창업 40s/인터뷰/ 아래에 11개 카테고리 폴더 (팀 미팅 11개, CEO 미팅 19개 등)가 존재. 총 800+ HTML 보관 중.

점진 마이그레이션 권장. 당장 일괄 이동하면 위험. 오늘부터 새 회의록만 앱 발행 경로 사용. 기존 분류 폴더는 그대로 두고, 발행 가치 있는 것만 골라 점진적으로 인박스 처리.

예상 절약: 회의록 1건당 수동 분류 시간 약 3분 → 0분. 월 30건 가정 시 월 90분 절약.

5오늘 박규호 부장이 확인할 것

  • 1️⃣ 데스크탑에 📁 지식창고_발행대기 폴더가 생성되었는지 확인
  • 2️⃣ 그 안에 ⚡발행대기처리.command 파일 더블클릭 권한 확인 (처음엔 "확인되지 않은 개발자" 경고 나올 수 있음 → 우클릭 → 열기)
  • 3️⃣ 영상회의록추출기 Rev.3 앱 안의 "발행하기" 버튼 위치 확인 (이미 코드에 존재)
  • 4️⃣ 슬랙 채널에 본 SOP 페이지 알림이 도착했는지 확인
